## 🤝 Code of Conduct

Before making any contribution, please read our [Code of Conduct](./CODE_OF_CONDUCT.md). We are committed to maintaining a welcoming and inclusive environment for everyone.

## How to Contribute

There are several ways to contribute to the project:

### 1. **Fix Bugs**
Found a problem? Open an issue or submit a pull request with a fix.

### 2. **Add Features**
Have an idea to improve the project? Please discuss it first by opening an issue before you start developing.

### 3. **Improve Documentation**
Documentation can always be improved. Contributions are welcome!

### 4. **Share Feedback**
Your feedback and suggestions help us improve the project.

## Contribution Process

### Preparing Your Development Environment

Check our [Developer Guide](./DEVELOPERS.md) to set up your local development environment.

### Git Workflow

1. **Fork** the repository
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/zats-sustainability-websites.git
   cd zats-sustainability-websites
   ```
3. **Create a branch** for your feature or bugfix:
   ```bash
   git checkout -b feature/my-new-feature
   # or
   git checkout -b fix/my-bugfix
   ```
4. **Commit** your changes with clear messages:
   ```bash
   git commit -m "feat: add new feature"
   git commit -m "fix: resolve issue #123"
   ```
5. **Push** to your fork:
   ```bash
   git push origin feature/my-new-feature
   ```
6. **Open a Pull Request** on the main repository

## 🐛 Reporting Bugs

### Before Reporting a Bug

- Check that the bug hasn't already been reported
- Make sure you're using the latest version
- Consult the documentation and closed issues

### How to Report a Bug

1. **Use a clear and descriptive title**
2. **Describe the exact steps to reproduce the problem**
3. **Provide specific examples** to illustrate the steps
4. **Describe the observed behavior** and what you expected
5. **Include screenshots or logs** if possible
6. **Mention your environment** (OS, browser, version, etc.)

## 💡 Suggesting Enhancements

1. **Use a clear and descriptive title**
2. **Provide a detailed description** of the enhancement
3. **List potential use cases**
4. **Mention any alternatives** you've considered
5. **Be patient** - review can take time

## Pull Requests

### Pull Request Guidelines

1. **One feature or bugfix per PR**
2. **Follow the existing code style** of the project
3. **Include tests** if applicable
4. **Update documentation** if necessary
5. **Write a clear title and description**

### Pre-Submission Checklist

- [ ] My code follows the project style
- [ ] I have tested the changes locally
- [ ] I have updated the documentation if necessary
- [ ] Changes don't introduce new warnings
- [ ] I respect the Code of Conduct

### Review Process

- At least one maintainer will review your PR
- Changes may be requested
- Once approved, your PR will be merged
